# BuyerLab AI Demo Script

Use this 60-second script for hackathon judging.

Before recording, set `BUYERLAB_MOCK_MODE=true` for a deterministic demo if a Gemini API key is not available.

## 0-10 sec: Problem

“E-commerce sellers often discover product-page problems after they already spent money on ads or launched to customers. BuyerLab AI helps identify those weaknesses before launch.”

## 10-20 sec: Product Input

“Here I enter a product, such as wireless headphones or running shoes. The page includes price, audience, value proposition, shipping, return policy, trust signals, reviews, CTA, and image notes.”

## 20-35 sec: AI Buyer Simulation

“BuyerLab AI simulates four buyer personas: Skeptic Buyer, Bargain Hunter, Impulsive Buyer, and Trust Seeker. Each one returns a structured decision, purchase intent score, reason, objections, missing information, and suggested fix.”

## 35-45 sec: Judge Report

“The Judge Agent reads the persona responses and debate history. It produces a simulated conversion score, buyer loss reasons, winning personas, lost personas, risk scores, and prioritized action items.”

Important wording:

“This is not a real market prediction. It is an AI-simulated buyer reaction score to help sellers find product-page weaknesses.”

## 45-55 sec: Attention/Friction Map and Fix Plan

“The dashboard then scores each product page section for AI-simulated buyer attention and conversion friction. This is not real eye-tracking. It highlights areas like price, trust signals, return policy, shipping, reviews, and CTA where buyers may hesitate.”

“The optimizer turns those findings into practical seller-focused copy improvements.”

## 55-60 sec: Before-After Improvement

“Finally, BuyerLab AI applies the optimized copy and simulates the page again. The before-after comparison shows the simulated conversion score delta, improved sections, remaining risks, and a concise summary.”

## Suggested Demo Product

Use one of these because they create understandable buyer objections:

- Wireless headphones
- Running shoes

Example angle:

“These wireless headphones sound promising, but the first simulation may expose missing warranty details, weak social proof, unclear shipping, or poor value justification. Then BuyerLab AI suggests specific fixes and re-runs the simulation.”

## Closing Line

“BuyerLab AI gives sellers a faster pre-launch diagnostic loop: simulate buyers, find conversion blockers, improve the page, and test again before spending money on traffic.”
